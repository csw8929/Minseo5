## Problem Statement
Minseo5의 현재 핵심 루프는 "문자 가져오기 → 파싱 → 리스트에 쌓기"다. 기록은 잘 되지만,
쌓인 데이터가 **리스트로만** 보여 "이번 달 어디에 썼는지"가 한눈에 안 들어온다.
기록이 통찰(트렌드)로 바뀌는 순간이 없고, 남에게 보여줄 만한 화면도 없다.

## What Makes This Cool
"눈에 보이는 소비 트렌드." 쌓인 기록이 **카테고리 도넛 + 일별 추이 + 사용처 TOP5**로
살아난다. 목표 청중은 **친구/남들(자랑용)** — 그래서 차트 자체가 "와" 포인트가 돼야 한다.
실용성보다 보기 예쁜 시각화에 무게를 둔다.

## Constraints
- "소스 안 건드리고 룰만 바꿔 확장" 철학 유지 (기존 rules.json 패턴 계승).
- 기존 import → parse → list 루프는 건드리지 않는다 (외과적 변경).
- 개발 단계라 DB 파괴적 마이그레이션 허용(`fallbackToDestructiveMigration`).
- 배포는 기존 채널(NAS 릴리스 + MinseoStore) 그대로.

## Premises (전부 동의됨)
1. 트렌드 화면 가치의 대부분은 기존 데이터(날짜·금액·사용처)만으로 나온다. 카테고리는
   "도넛 차트" 하나를 위한 추가 재료다.
2. 카테고리는 DB에 저장하지 않고 **표시 시점에 룰로 계산**한다. rules.json만 고치면
   과거 기록도 자동 재분류된다. 스키마 변경 불필요.
3. 분류 룰은 파싱 룰(`rules`)과 **별개 섹션**(`categories`)이다. 파싱 룰에 category를
   박으면 카드사 단위로만 묶여 무의미하므로, 사용처 텍스트 → 카테고리 매핑을 새로 둔다.
4. 차트는 검증된 라이브러리 **MPAndroidChart**로 그린다 (자랑용 퀄리티를 싸게 획득).
5. 새 화면 + 분류 유틸 + 룰 섹션만 추가하고 기존 루프는 보존한다.

## Approaches Considered
### Approach A: 라이브러리 없는 한 화면 (minimal viable)
새 StatsActivity 하나. 카테고리/TOP5는 색깔 View width 막대로, 합계·전월대비는 텍스트.
의존성 0. 가장 작은 diff지만 도넛이 없어 자랑용 임팩트가 약함. (human ~1일 / CC ~20분)

### Approach B: MPAndroidChart 통계 화면 (ideal architecture) — 채택
새 StatsActivity + MPAndroidChart. 도넛(카테고리 비중) + 일별 추이 바 + 사용처 TOP5
+ 합계·전월대비 카드. rules.json `categories` 섹션 추가, 분류는 표시 시점 계산.
(human ~3~4일 / CC ~45~60분, Risk Med)

### Approach C: B + 공유 카드 생성 (creative/lateral, fast-follow)
B 위에 "이미지로 공유" 버튼. 도넛+TOP3+이번달 총액+위트 한 줄을 비트맵으로 렌더해
ACTION_SEND. 들어오는 공유(import)의 거울 — 나가는 공유. "자랑"의 진짜 형태지만 범위가
가장 큼. (human ~1주 / CC ~90분)

## Recommended Approach
**Approach B.** "보이는 소비 트렌드 + 자랑용"의 핵심은 카테고리 도넛이고, B가 그걸 가장
적은 노력으로 제대로 준다. C(공유 카드)는 B 위에 얹는 후속 작업으로 둔다.

### 기술 설계

**1. rules.json 확장 (version 3)**
기존 RuleStore의 버전 기반 reseed 메커니즘을 그대로 활용 — version만 올리면 단말의
구버전 rules.json이 자동 갱신된다.
```json
{
  "version": 3,
  "yearInference": "recentPast",
  "rules": [ /* 기존 파싱 룰 그대로 */ ],
  "categories": [
    { "category": "카페", "keywords": ["스타벅스","이디야","투썸","메가","컴포즈","빽다방"] },
    { "category": "식비", "keywords": ["배달의민족","요기요","마트","편의점","GS25","CU","김밥"] },
    { "category": "교통", "keywords": ["택시","카카오T","지하철","버스","주유","에너지","칼텍스"] },
    { "category": "쇼핑", "keywords": ["쿠팡","올리브영","무신사","다이소"] }
  ],
  "defaultCategory": "기타"
}
```

**2. Categorizer 유틸 (신규)**
- `Categorizer.of(context)` — rules.json의 `categories`/`defaultCategory`를 1회 로드.
- `categorize(purpose)` — 사용처 문자열에 키워드가 substring 매칭되면 그 카테고리, 없으면
  defaultCategory. (정규식 지원은 키워드를 정규식으로 해석하면 무료로 확장 가능.)
- 월별 집계 시점에 각 레코드의 `purpose`에 적용 → 카테고리별 합계 산출.

**3. 집계**
데이터셋이 작고 `allowMainThreadQueries()`가 켜져 있으므로, 선택 월 레코드를 로드해
메모리에서 집계한다(카테고리별 합/일별 합/사용처별 합). 별도 복잡 SQL 불필요.
필요 시 SpendingDao에 `selectByMonth(yyyyMM)` 정도만 추가.

**4. StatsActivity (신규)**
- 진입점: MainActivity 툴바 메뉴 "통계" 항목.
- 월 네비게이션은 MainActivity 패턴 재사용(선택 월을 intent로 전달 또는 자체 네비).
- 구성:
  - 헤더 카드: 이번 달 합계 + 전월 대비(증감액/화살표; 전월 데이터 없으면 "—").
  - PieChart(도넛): 카테고리 → 합계.
  - BarChart: 일(1..말일) → 합계 (일별 추이; 주말 강조 고려).
  - TOP5: 사용처별 합계 상위 5 (RecyclerView 또는 정적 행).
- 색 팔레트: 카테고리별 고정 색(식비 주황 / 카페 갈색 / 교통 파랑 …). 기존 accent와 조화.

**5. 의존성**
- `settings.gradle.kts` repositories에 JitPack 추가: `maven { url = uri("https://jitpack.io") }`
- `com.github.PhilJay:MPAndroidChart:v3.1.0`
